API Документация

1. Процесс авторизации

Шаг 1: Пациент переходит на страницу авторизации:
{uiBasePath}/auth/{patentKey}/{patientCode}

2. Авторизация

POST
{apiBasePath}/auth

Тип ответа: json

Тело запроса:
{ "patentKey": "{patentKey}", "patientCode": "{patientCode}" }
Успешный ответ:
{ "JWT": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." }
Ошибка:
{ "status": "error", "message": "Неверный код пациента" }
Важно: После успешной авторизации добавьте JWT токен в заголовок всех последующих запросов:
Authorization: Bearer {JWT}

3. Получение документов

POST
{apiBasePath}/doc

Тип ответа: arraybuffer

Тело запроса:
{ "action": "getPackage" | "getPdp" | "getEdm" }
Успешный ответ:
<arraybuffer> // Бинарные данные документа
Ошибка:
{ "status": "error", "message": "Документ не найден" }

4. Подтверждение документов

POST
{apiBasePath}/approve

Тип ответа: json

Тело запроса:
{ "type": "pdp" | "edm" }
Успешный ответ:
{ "status": "success", "message": "Документ успешно подтвержден" }
Ошибка:
{ "status": "error", "data": null, "message": "Ошибка подтверждения документа" }

5. Подписание документов

POST
{apiBasePath}/sign

Тип ответа: json

Тело запроса:
{ "action": "onSigning" | "onReject" }
Успешный ответ:
{ "status": "success", "message": "Документ успешно подписан" }
Ошибка:
{ "status": "error", "message": "Ошибка подписания документа" }
